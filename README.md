### 機能

DNSPacket : DNSパット(クラス)
  変数(public):
    Header (H)          : Headerセクション(構造体)。
    Question (Q)        : Questionセクション(構造体)。
    ResourceRecord (RR) : Answer,Authority,Additionalセクション(構造体)の一部。
    octets              : オクテット列。
    (各構造体の変数には初期値(0x00など)が設定されています。)
  関数(public):
    各setter()              : header_setterだけ、idを指定できるようになっている。
                              0以上65535以下の範囲なら、指定した値になる。そうでなければ、予測不可な乱数を生成する。
    各getter()              : 特筆すべきことなし。
    parameter_to_octets()   : 構造体内のいろんな型のパラメータからオクテット列へ変換する関数。
    octets_to_parameter()   : オクテット列から各パラメータに変換する関数。
      ・オクテットから読み取って、クラス内の変数に値が保存される。
      ・オクテット列が過度に小さくても(つまり構造的に不正パケットでも)、特にエラーを出さず変換できるところまで変換する。
      ・圧縮ポインタに対応済。不正だと検知した場合、その時点で処理を中断し、以後のいかなる処理もしない。
        ・自分以後のポインタを指していたら、そのポインタを含めて値に格納し中断する。
        ・ポインタの遷移回数がlimit(デフォルト:16)を超えると、その時点で値を含めずに中断する。
        ・ラベル長が63を超えると、64を末尾に追加し中断する。
        ・ラベル長だけ文字が存在しないと、その時点で中断する。
        ・ドメイン名の大きさが255を超えると、末尾にその大きさを追加し中断する。
    fqdn_string_to_octets() : FQDN(string)からオクテット列に変換する関数。
      ・使用必須ではない。オクテット列なんてよくわからんという人向け。
      ・間違えてwww.example.comを渡しても、www.example.com.に変換されるようになっている。
      ・ラベル長が64以上だったり、引数が空文字だったり、オクテット列の長さが255を超えるようだと、{}を返す。
        何が間違っていたかは確認するすべはないので、引数の文字列とにらめっこして特定してね。(多分すぐわかるはず。)
    fqdn_octets_to_string() : オクテット列からFQDN(string)に変換する関数。
    print_parameter()       : 各パラメータを標準出力する関数。

### 最後に
  値の変換はドメイン名←→オクテット列しか用意していません。
  レコードタイプなどの変換も用意してもいいんですが、調べてくれる方が個人的にはよいのかなと。
